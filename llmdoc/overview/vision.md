# 项目愿景与目标

## 核心理念

**"记忆保留痕迹，痕迹提供价值"**

在数字化生存的时代，知识工作者每天处理数千兆字节的视觉与文本信息——代码片段、即时通讯、文档草稿、浏览记录——然而这些高价值的"痕迹"(Traces)大多在窗口关闭的瞬间便烟消云散。

Engram 的目标是通过技术手段跨越这一鸿沟，构建用户的**数字孪生(Digital Twin)**。

## 项目名称含义

**Engram** (印迹) - 神经科学术语，指记忆在大脑中留下的物理痕迹。项目名称寓意将用户的数字活动转化为可检索、可分析的持久记忆。

## 长期愿景

### 第一阶段：全知之眼
- 无感知地记录用户的屏幕活动
- 高效提取文本与视觉信息
- 建立可搜索的时序记忆库

### 第二阶段：深度认知
- 语义理解屏幕内容
- 支持自然语言搜索
- 智能过滤敏感内容

### 第三阶段：记忆合成
- 周期性生成工作摘要
- 提取实体关系构建知识图谱
- 与 AI Agent 生态集成 (MCP)

### 第四阶段：生态扩展
- 插件系统支持社区扩展
- 多设备记忆同步(可选)
- 高级分析与可视化

## 设计原则

### 1. 本地优先 (Local First)
所有数据处理与 AI 推理在本地完成，绝不上传用户数据。

### 2. 隐私至上 (Privacy by Design)
- 语义黑名单过滤敏感内容
- 数据库加密存储
- 精细的权限控制

### 3. 无感运行 (Invisible Computing)
- CPU 占用 < 5%
- 内存占用 < 500MB
- 电池友好

### 4. 开放互联 (Interoperability)
- MCP 协议支持
- WASM 插件系统
- 标准化数据格式

## 目标用户

1. **知识工作者** - 需要回顾研究材料、找回遗忘的信息
2. **程序员** - 搜索代码片段、追溯调试过程
3. **内容创作者** - 管理参考素材、追踪灵感来源
4. **AI 用户** - 为 Claude/Cursor 提供长期记忆上下文
